# 딥리서치 요청문 생성기

## 목적
새 테스트를 위한 딥리서치 요청문을 자동 생성합니다.
사용자가 복사해서 ChatGPT/Gemini/Perplexity에 붙여넣기 할 수 있는 완성된 프롬프트를 제공합니다.

## 호출 예시
```
"whiskey 테스트 리서치 요청문 만들어줘"
"고양이 품종 매칭 딥리서치 프롬프트 생성해"
```

## 템플릿 참조
`docs/test-creation/RESEARCH_REQUEST.md` 파일의 템플릿을 기반으로 생성합니다.

## testType별 템플릿

### personality (성격 테스트)
```markdown
# {subject} 성격 테스트 리서치 요청

## 배경
{subject}의 성격/행동 유형을 분석하는 테스트를 만들려고 합니다.

## 요청 사항

### 1. 학술적 프레임워크
{subject} 성격 분석에 사용되는 학술적 프레임워크를 찾아주세요.
- 기존 연구/논문에서 사용된 차원
- 각 차원의 정의와 측정 방법
- 신뢰도/타당도 검증 여부

### 2. 차원 설계
5-6개 차원을 제안해주세요. 각 차원에 대해:
- 키(영문 camelCase)
- 이름(한글 2-4자)
- 이모지(1개)
- 설명(10-20자)
- LOW ↔ HIGH 양극단

### 3. 결과 유형
8-16개 성격 유형을 제안해주세요. 각 유형에 대해:
- 이름
- 이모지
- 한줄 설명
- 조건 (어떤 차원이 high/low일 때)
- 상세 해석

### 4. 질문 예시
각 차원당 2-3개 질문 예시 (5/3/1 점수 선택지 포함)

## 출력 형식
마크다운 테이블 형식으로 정리해주세요.
```

### matching (매칭 테스트)
```markdown
# {subject} 매칭 테스트 리서치 요청

## 배경
사용자 취향에 맞는 {subject}를 추천하는 테스트를 만들려고 합니다.

## 요청 사항

### 1. 분류 체계
{subject}의 업계/학술 분류 체계를 찾아주세요.
- 주요 카테고리/유형
- 각 유형의 특성
- 분류 기준

### 2. 선호도 차원
4-6개 선호도 차원을 제안해주세요:
- 키(영문 camelCase)
- 이름(한글 2-4자)
- 이모지(1개)
- 설명(10-20자)
- LOW ↔ HIGH

### 3. 추천 결과
8-12개 추천 결과를 제안해주세요:
- 이름
- 이모지
- 한줄 설명
- 조건 (어떤 차원이 high/low일 때 추천)
- 추천 포인트 3개

### 4. 질문 예시
각 차원당 2-3개 질문 예시 (5/3/1 점수 선택지 포함)

## 출력 형식
마크다운 테이블 형식으로 정리해주세요.
```

### situation (상황 대처 테스트)
```markdown
# {subject} 상황 대처 테스트 리서치 요청

## 배경
특정 상황에서의 대처 방식을 분석하는 테스트를 만들려고 합니다.

## 요청 사항

### 1. 학술적 프레임워크
{subject} 관련 대처 방식 연구를 찾아주세요.
- 기존 연구에서 사용된 대처 유형
- 각 유형의 정의
- 상황별 적합한 대처 방식

### 2. 대처 차원
4-6개 대처 차원을 제안해주세요.

### 3. 대처 유형
6-10개 대처 유형을 제안해주세요.

### 4. 상황 질문
각 차원당 2개 상황 질문 예시

## 출력 형식
마크다운 테이블 형식으로 정리해주세요.
```

## 생성 프로세스

1. 사용자 입력에서 subject와 testType 파악
2. 해당 템플릿 선택
3. {subject} 치환
4. 완성된 프롬프트 출력

## 출력 형식

```
📋 {subject} 딥리서치 요청문

아래 내용을 ChatGPT/Gemini/Perplexity에 복사해서 붙여넣으세요:

---
[생성된 프롬프트]
---

💡 팁:
- ChatGPT: "Deep Research" 모드 사용 권장
- Gemini: "딥 리서치" 기능 사용
- Perplexity: Pro 모드 + "Academic" 필터

결과를 받으면 `research/{subject}.md`에 저장하세요.
```

## ⚠️ 강제 중단 조건 (필수!)

**아래 상황에서는 즉시 작업 중단하고 사용자에게 보고:**

| 상황 | 대응 |
|------|------|
| testType 파악 불가 | 사용자에게 personality/matching/situation 중 선택 요청 |
| 요청한 주제가 너무 광범위 | 범위 좁히기 제안 (예: "음식" → "라면" 또는 "커피") |
| 템플릿 참조 파일 없음 | 중단 + 내장 템플릿으로 대체 |

**생성 시 확인:**
- 템플릿 형식 정확히 따르기
- testType에 맞는 수치 기준 포함
- 모든 필수 섹션(차원, 결과, 질문) 요청 포함

**절대 금지:**
- 불완전한 프롬프트 제공
- testType 없이 기본값 사용
- 사용자가 요청하지 않은 다른 작업 수행
- 필수 섹션 누락된 채로 출력
