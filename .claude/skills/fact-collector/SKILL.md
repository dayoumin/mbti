# 팩트 수집 스킬

## 목적
퀴즈/투표 콘텐츠 생성 시 필요한 검증된 팩트를 웹검색으로 수집하고 저장합니다.

## 팩트 필요 카테고리

| 카테고리 | 팩트 필수 | 검색 키워드 예시 |
|----------|----------|-----------------|
| cat | O | "고양이 정상 체온 수의학", "cat normal temperature veterinary" |
| dog | O | "강아지 호흡수 정상 범위", "dog normal respiratory rate" |
| rabbit | O | "토끼 이빨 성장 속도" |
| hamster | O | "햄스터 수명 평균" |
| plant | O | "산세베리아 CAM 식물", "monstera fenestration reason" |
| coffee | O | "아메리카노 카페인 함량" |
| alcohol | O | "위스키 종류별 특성" |

## 수집 프로세스

### 1단계: 팩트 필요 여부 확인
```
요청된 카테고리가 팩트 필요 카테고리인가?
├─ 예 → 2단계로
└─ 아니오 → 팩트 수집 불필요, 종료
```

### 2단계: 기존 팩트 확인
```bash
# 기존 팩트 파일 확인
cat research/facts/{category}.md

# 필요한 팩트가 이미 있는가?
├─ 있음 → 해당 팩트 반환
└─ 없음 → 3단계로
```

### 3단계: 웹검색으로 팩트 수집
```
1. 신뢰할 수 있는 출처 우선 검색
   - 수의학: AAHA, AAFP, Merck Vet Manual, VCA Hospitals
   - 식물학: RHS, NASA Clean Air Study
   - 일반: 학술 논문, 공식 기관 사이트

2. 검색어 구성
   - 한글 + 영문 병행 검색
   - "정확한 수치" + "출처" 키워드 포함

3. 교차 검증
   - 2개 이상 출처에서 일치하는 정보 확인
   - 불일치 시 더 신뢰할 수 있는 출처 채택
```

### 4단계: 팩트 파일에 저장
```markdown
## {category}-fact-{번호}: {제목}
- **값**: {구체적 수치/사실}
- **출처**: {출처명}
- **URL**: {URL}
- **검증일**: {YYYY-MM-DD}
- **사용된 콘텐츠**: (콘텐츠 생성 후 추가)
- **비고**: {추가 설명}
```

## 출력 형식

```
📚 팩트 수집 완료

## 수집된 팩트
| ID | 제목 | 값 | 출처 |
|----|------|-----|------|
| {id} | {제목} | {값} | {출처} |

## 저장 위치
research/facts/{category}.md

## 사용 방법
콘텐츠 생성 시 source: "{fact-id}" 추가 (예: source: 'cat-fact-001')

## 참고
상수 정의: `src/data/content/fact-constants.ts`
```

## 신뢰도 기준

| 출처 유형 | 신뢰도 | 예시 |
|----------|--------|------|
| 학술 논문/저널 | 최상 | PubMed, Journal of Veterinary Medicine |
| 공식 기관 | 상 | AAHA, AAFP, AKC, RHS, NASA |
| 전문 병원/클리닉 | 상 | VCA Hospitals, 대학병원 |
| 전문 매체 | 중 | Scientific American, 전문 잡지 |
| 일반 웹사이트 | 하 | 블로그, 커뮤니티 (교차검증 필수) |

## 불확실한 정보 처리

1. **출처를 찾을 수 없음** → 팩트 추가 안 함, 사용자에게 알림
2. **출처 간 수치 불일치** → 더 신뢰할 수 있는 출처 채택 + 범위로 표기
3. **최신 정보 필요** → 검색 시 연도 필터 적용 (최근 5년)

## 사람 개입 요청

다음 상황에서는 사용자에게 확인 요청:

| 상황 | 메시지 |
|------|--------|
| 신뢰할 수 있는 출처 없음 | "'{주제}'에 대한 신뢰할 수 있는 출처를 찾지 못했습니다. 직접 확인하시겠습니까?" |
| 출처 간 큰 불일치 | "'{주제}'에 대해 출처마다 다른 정보가 있습니다: A(출처1) vs B(출처2). 어느 것을 사용할까요?" |
| 의료/법적 정보 | "이 정보는 전문가 확인이 권장됩니다. 그대로 사용할까요?" |
