# 인사이트 시스템 딥리서치 요청

> 다른 AI(ChatGPT, Gemini, Perplexity)에게 보낼 딥리서치 프롬프트
> 작성일: 2024-12-25

---

## 연구 배경

테스트/퀴즈/투표 참여 기반 성격 분석 앱의 **인사이트 시스템**을 설계 중입니다.
핵심 차별점은 "AI 채팅에서 말한 것"이 아닌 **"앱에서 행동한 것"을 분석**하는 것입니다.

### 설계 개요
- **7단계 점진적 해금**: 활동량에 따라 인사이트 단계적 공개
- **3단계 복잡도**: 단순 집계(무료) → 60개 룰 매칭(무료) → AI 생성(유료)
- **태그 시스템**: 투표 선택/테스트 결과에 성격 태그 부여
- **DB**: Turso(SQLite) + JSON 컬럼

---

## 연구 주제 6가지

### 1. 심리학적 근거 검증

**질문:**
- Big Five(OCEAN) 성격 모델의 과학적 타당성과 최신 연구(2023-2024)
- Thomas-Kilmann TKI 갈등 유형 도구의 신뢰도/타당도
- 반려동물 선호(개 vs 고양이)와 성격 특성의 상관관계 연구
- 커피/차 선호와 성격의 상관관계 연구 (있다면)
- 이러한 심리학 모델을 앱 태그 시스템에 적용할 때 주의점

**원하는 결과:**
- 각 모델의 신뢰도 수치 (Cronbach's alpha, test-retest 등)
- 주요 연구 논문/메타분석 출처
- 태그 시스템 설계에 반영할 구체적 권장사항

---

### 2. 점진적 해금 + 게이미피케이션 UX

**질문:**
- Progressive Disclosure가 사용자 리텐션에 미치는 영향
- 성격 테스트/퀴즈 앱의 게이미피케이션 성공 사례
- "N개 더 하면 해금" 유도 메시지의 효과적인 패턴
- 해금 조건 설정 시 최적의 난이도 곡선
- Spotify Wrapped, YouTube Recap 등 연말결산 기능의 참여율 데이터

**원하는 결과:**
- 리텐션 증가율 수치가 있는 사례
- 해금 단계별 권장 활동량 (우리는 7단계 계획)
- 피해야 할 안티패턴

---

### 3. TypeScript/JavaScript 룰 엔진

**질문:**
- JSON 기반 조건 처리 룰 엔진 라이브러리 비교:
  - json-rules-engine
  - json-rules-engine-simplified
  - RulePilot
  - node-rules
- 각 라이브러리의 장단점, 성능, 번들 크기
- 60개 정도의 룰을 처리할 때 커스텀 구현 vs 라이브러리 사용 비교
- 룰 우선순위, 부분 매칭 등 고급 기능 지원 여부

**원하는 결과:**
- 라이브러리 선택 권장사항
- 커스텀 구현 시 참고할 패턴
- 성능 벤치마크 (있다면)

---

### 4. Turso(SQLite) JSON 쿼리 최적화

**질문:**
- SQLite JSON 컬럼 쿼리 성능 최적화 방법
- 태그 시스템 스키마 설계: JSON 배열 vs 별도 테이블(Many-to-Many)
- Generated Column + Index 패턴의 실제 성능
- 태그 기반 검색 시 최적의 인덱싱 전략
- Turso 특화 고려사항 (edge 배포, 복제 등)

**원하는 결과:**
- 스키마 설계 권장안 (우리 유스케이스: 60개 룰, 100개 결과, 50개 투표)
- 쿼리 예시 코드
- 성능 주의사항

---

### 5. LLM 비용 최적화

**질문:**
- 성격 분석 리포트 생성 시 프롬프트 최적화 기법
- 토큰 수 줄이는 구체적 방법 (압축, 구조화 등)
- Claude Haiku vs Sonnet vs GPT-4o-mini 비용/품질 비교
- 캐싱 전략 (유사 프로필 캐싱 가능?)
- 월 1만 사용자 기준 예상 비용 계산

**원하는 결과:**
- 비용 35-80% 절감 가능한 구체적 기법
- 프롬프트 템플릿 최적화 예시
- 모델 선택 가이드

---

### 6. 비즈니스 모델 & 전환율

**질문:**
- 16Personalities, Truity 등 성격 테스트 서비스의 수익 모델
- 무료 테스트 → 유료 리포트 전환율 벤치마크
- 프리미엄 vs 구독 모델 장단점
- 심리/웰니스 앱의 평균 LTV, ARPU
- AI 분석을 유료로 제공할 때 적정 가격대

**원하는 결과:**
- 전환율 벤치마크 (3-5% 좋음, 6-8% 훌륭 등)
- 가격 책정 참고 데이터
- 수익화 타이밍 권장사항

## 프로젝트 컨텍스트

### 현재 앱 상태
- 33개 테스트 (성격, 반려동물, 라이프스타일 등)
- 50+ 투표, 30+ 퀴즈
- 기술 스택: Next.js + TypeScript + Turso

### 인사이트 7단계 (계획)
1. 기본 성향 (테스트 1개)
2. 성격 조합 (테스트 3개)
3. 판단 스타일 (투표 10개)
4. 관심사 지도 (활동 15개)
5. 관계 패턴 (관계 활동 10개)
6. 숨은 패턴 (활동 30개)
7. AI 종합 분석 (유료)

### 태그 시스템 (계획)
```typescript
// 성격 태그
['extroverted', 'introverted', 'logical', 'emotional',
 'planned', 'spontaneous', 'independent', 'collaborative']

// 결정 태그
['practical', 'emotional', 'safe', 'adventurous',
 'solo', 'together', 'direct', 'indirect']

// 관계 태그
['expressive', 'reserved', 'confrontational', 'avoidant',
 'close-bonding', 'space-needing']
```
