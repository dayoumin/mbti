# 운세 콘텐츠 기획을 위한 딥리서치 요청

> **목적**: 퀴즈/투표/커뮤니티 앱에 통합할 운세 콘텐츠 기획
> **제약**: 의료/금융/법률 조언 금지, 엔터테인먼트 목적

---

## 연구 질문 1: 운세 콘텐츠 유형 비교

### 질문
"모바일 앱에서 활용 가능한 운세 콘텐츠 유형(별자리, 띠, 타로, 사주, 수비학 등)을 비교 분석해줘. 각 유형별로:

1. **콘텐츠 생성 난이도** (필요한 기초 지식의 양과 복잡도)
2. **대중 친숙도** (한국 2030세대 기준)
3. **게이미피케이션 적합성** (퀴즈, 투표, 수집 요소와의 결합 가능성)
4. **법적 리스크** (과학적 검증 불가 주장으로 인한 문제 가능성)
5. **차별화 포인트** (기존 운세 앱과 다르게 할 수 있는 요소)

비교 표와 함께 **MVP로 추천하는 유형 1-2개**와 그 이유를 설명해줘."

## 연구 질문 2: 별자리 운세 콘텐츠 설계

### 질문
"엔터테인먼트 앱을 위한 별자리(황도 12궁) 운세 콘텐츠를 설계하려고 해. 다음 내용을 정리해줘:

### A. 12별자리 기본 정보
각 별자리별로:
- 날짜 범위
- 원소(불/땅/공기/물)와 특성
- 수호 행성
- 핵심 성격 키워드 5개
- 긍정적 특성 3개
- 주의할 점 3개 (부정적이 아닌 '성장 포인트'로 표현)

### B. 운세 메시지 패턴
- 오늘의 운세에 사용할 수 있는 **긍정적 메시지 템플릿** 20개
- 연애/재물/건강/직장 카테고리별 **안전한 표현** 예시 각 5개
- **피해야 할 표현**과 **대안 표현** 비교 10쌍

### C. 게이미피케이션 아이디어
- 별자리 퀴즈 문제 예시 10개
- 별자리 궁합 투표 아이디어 5개
- 수집/달성 시스템 아이디어 3개

**중요**: 모든 표현은 '~할 수 있다', '~하는 경향이 있다' 등 개연성 어미 사용, 단정적 예언 금지"

## 연구 질문 3: 띠(12지신) 운세 콘텐츠 설계

### 질문
"한국형 운세 앱을 위한 띠(12지신) 콘텐츠를 설계하려고 해. 다음 내용을 정리해줘:

### A. 12띠 기본 정보
각 띠별로:
- 해당 연도 계산법 (2025년 기준 나이 포함)
- 오행(목화토금수) 연결
- 핵심 성격 키워드 5개
- 긍정적 특성 3개
- 성장 포인트 3개

### B. 띠별 궁합 매트릭스
- 잘 맞는 띠 조합 (삼합, 육합)
- 주의가 필요한 조합 (긍정적으로 표현)
- 궁합 설명 시 **피해야 할 표현** (예: '상극', '불화') → **대안 표현**

### C. 2025년 을사년 테마
- 뱀띠 해의 전체적인 기운/테마 (긍정적으로)
- 각 띠별 2025년 키워드 1-2개

### D. 게이미피케이션 아이디어
- 띠 맞추기 퀴즈 (나이/연도 기반) 5개
- 띠별 투표 아이디어 5개
- 띠 조합 커뮤니티 이벤트 아이디어 3개

**중요**: '흉', '재앙', '피해야 함' 등의 표현 금지, 모든 내용을 긍정적/성장 관점으로 작성"

## 연구 질문 4: 간단 타로 시스템 설계

### 질문
"모바일 앱용 간소화된 타로 시스템을 설계하려고 해. 78장 전체가 아닌 **메이저 아르카나 22장**만 사용:

### A. 22장 카드 정보
각 카드별로:
- 카드 이름 (한글/영문)
- 카드 번호
- 핵심 키워드 3개
- 정방향 의미 (긍정적, 2-3문장)
- 역방향 의미 (부정적이 아닌 '주의/성찰 포인트'로, 2-3문장)
- 조언 메시지 1문장

### B. 간단 배열법
- 원카드 리딩 (오늘의 카드)
- 쓰리카드 리딩 (과거/현재/미래)
- 각 배열법의 해석 가이드

### C. 앱 UX 아이디어
- 카드 뽑기 인터랙션 아이디어 3개
- 카드 수집/도감 시스템 아이디어
- 카드별 이미지 스타일 추천 (저작권 안전한 방향)

### D. 안전한 해석 원칙
- 절대 사용 금지 표현 10개
- 권장 표현 패턴 10개
- '나쁜 카드'를 긍정적으로 해석하는 예시 5개

**중요**: 타로는 '자기 성찰 도구'로 포지셔닝, 예언/점술이 아닌 '오늘의 영감' 콘셉트"

## 연구 질문 5: 운세 콘텐츠 MVP 로드맵

### 질문
"운세 콘텐츠를 퀴즈/투표/커뮤니티 앱에 통합하려고 해. 다음 조건으로 MVP 로드맵을 제안해줘:

### 조건
- 개발 리소스: 1인 (AI 코딩 지원)
- 기존 기능: 성격 테스트, 퀴즈, 투표, 커뮤니티
- 목표: 일일 방문 유도, 체류 시간 증가, 바이럴

### 요청 사항

**Phase 1 (MVP)**
- 가장 빠르게 출시할 수 있는 운세 기능 1개
- 필요한 콘텐츠 양 (결과 수, 메시지 수)
- 기존 기능과의 연결 방법
- 예상 개발 범위

**Phase 2 (확장)**
- 추가할 운세 유형 1-2개
- 게이미피케이션 강화 방안
- 커뮤니티 연동 아이디어

**Phase 3 (고도화)**
- 개인화/추천 시스템
- 유료화 가능 요소
- 장기 리텐션 전략

각 Phase별 핵심 지표(KPI)와 성공 기준도 제안해줘."

## 연구 질문 6: 경쟁 서비스 분석

### 질문
"한국에서 인기 있는 운세/타로 앱 및 서비스를 분석해줘:

### 분석 대상
- 운세 전문 앱 (점신, 신한운세, 오늘의 운세 등)
- 포털 운세 서비스 (네이버, 다음)
- 커뮤니티 기반 (에브리타임 운세, 인스타 타로 등)

### 분석 항목
각 서비스별로:
1. 주요 기능과 콘텐츠 유형
2. 수익 모델 (광고/인앱결제/구독)
3. 차별화 포인트
4. 한계점 또는 개선 가능 영역
5. 우리가 배울 점

### 시장 기회
- 기존 서비스들이 놓치고 있는 영역
- '퀴즈+운세', '투표+운세', '커뮤니티+운세' 결합의 차별점
- 2030 타겟에게 어필할 수 있는 포지셔닝

**비교 표 형태로 정리해줘.**"
