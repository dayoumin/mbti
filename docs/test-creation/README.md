# 테스트 생성 시스템

> 100개 이상의 테스트를 일관된 품질로 만들기 위한 표준화된 워크플로우

---

## 핵심 워크플로우

```
┌─────────────────────────────────────────────────────────────────┐
│                    Test Creation Pipeline                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  [1단계] 딥리서치 요청                                            │
│  ┌────────────────────────────────────────┐                     │
│  │ RESEARCH_REQUEST.md 템플릿 복사        │                     │
│  │         ↓                              │                     │
│  │ ChatGPT/Perplexity에 요청              │                     │
│  │         ↓                              │                     │
│  │ 학술 근거 + 분류 체계 수집              │                     │
│  └────────────────────────────────────────┘                     │
│                      ↓                                           │
│  [2단계] 결과 정리                                               │
│  ┌────────────────────────────────────────┐                     │
│  │ RESEARCH_RESULT.md 포맷에 맞게 정리    │                     │
│  │         ↓                              │                     │
│  │ research/{subject}.md 저장             │                     │
│  └────────────────────────────────────────┘                     │
│                      ↓                                           │
│  [3단계] 테스트 생성 (AI)                                        │
│  ┌────────────────────────────────────────┐                     │
│  │ Claude Code에 요청:                    │                     │
│  │ "research/whiskey.md 기반으로          │                     │
│  │  테스트 생성해줘"                       │                     │
│  │         ↓                              │                     │
│  │ GENERATION_GUIDE.md 규칙 따라 생성     │                     │
│  │         ↓                              │                     │
│  │ 자동 검증 + 수정                        │                     │
│  └────────────────────────────────────────┘                     │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 빠른 시작

### 새 테스트 만들기

1. **딥리서치 요청**
   ```
   [RESEARCH_REQUEST.md](./RESEARCH_REQUEST.md) 템플릿을 ChatGPT에 복사
   ```

2. **결과 정리**
   ```
   [RESEARCH_RESULT.md](./RESEARCH_RESULT.md) 포맷으로 정리
   → research/{subject}.md 저장
   ```

3. **테스트 생성**
   ```
   Claude Code에게:
   "research/{subject}.md 기반으로 테스트 생성해줘"
   ```

### 기존 테스트 개선하기

1. [EXISTING_TESTS_AUDIT.md](./EXISTING_TESTS_AUDIT.md)에서 우선순위 확인
2. 해당 테스트의 리서치 진행
3. 개선 후 검증

---

## 품질 기준 요약

| 항목 | 기준 | 필수 여부 |
|------|------|----------|
| 차원 개수 | 5-6개 | 필수 |
| 질문 개수 | 12-16개 | 필수 |
| 결과 개수 | 8-16개 | 필수 |
| 중간점수(3) 포함 | 40%+ 질문 | 권장 |
| 결과 조건 | 2-3개씩 | 필수 |
| 도달 가능성 | 100% | 권장 |
| 리서치 근거 | 있음 | 권장 |
