# 딥리서치 요청 템플릿

> 이 템플릿을 ChatGPT, Perplexity 등 딥리서치 도구에 복사해서 사용하세요.
> 결과는 `research/{subject}.md`에 저장합니다.

---

## 사용 방법

1. 아래 템플릿 중 적절한 것을 선택
2. `{placeholder}`를 실제 값으로 교체
3. 딥리서치 도구에 붙여넣기
4. 결과를 [RESEARCH_RESULT.md](./RESEARCH_RESULT.md) 형식으로 정리
5. `research/{subject}.md`에 저장

---

## 템플릿 A: 성격 테스트 (Personality Test)

> 사람/동물의 성격 유형을 분석하는 테스트용

```
[주제]: {subject_name} 성격 테스트
(예: 사람 성격, 고양이 성격, 강아지 성격)

## 목적
이 주제로 **성격 유형 테스트**를 만들려고 합니다.
사용자가 질문에 답하면, 여러 성격 유형 중 하나로 분류됩니다.

## 요청 사항

### 1. 학술적 성격 모델
- 이 주제에 적용 가능한 검증된 성격 모델이 있나요?
  - 사람: Big Five (OCEAN), MBTI, HEXACO 등
  - 고양이: Feline Five (MCPQ-R)
  - 강아지: C-BARQ, Canine Big Five
- 각 모델의 차원(요인)과 정의
- 신뢰도/타당도가 검증된 연구

### 2. 차원(Dimension) 설계
5-6개 차원을 제안해주세요. 각 차원에 대해:
- **키(영문)**: camelCase, 예: openness, friendliness
- **이름(한글)**: 2-4자, 예: 개방성, 친화력
- **이모지**: 1개, 예: 🌟, 🤝
- **설명**: 10-20자, 이 차원이 무엇을 측정하는지
- **양극단**: LOW는 어떤 특성? HIGH는 어떤 특성?
- **근거**: 어떤 연구/모델에서 왔는지

### 3. 결과 유형 (8-16개)
각 성격 유형에 대해:
- **이름**: 3-8자
- **이모지**: 1개
- **한줄 설명**: 15-30자
- **조건**: 어떤 차원이 high/medium/low일 때 이 유형인지 (2-3개 조건)
- **특징**: 이 유형의 성격 특성
- **해석**: 50-100자 심층 분석
- **가이드**: 50-100자 조언/팁
- **분위기(mood)**: happy, cool, excited, calm 중 하나
- **색상 제안**: 이 유형에 어울리는 색상 (예: amber, rose, blue)

### 4. 질문 예시
각 차원당 2-3개 질문 예시를 제안해주세요:
- 질문 내용 (물음표로 끝나게)
- 3개 선택지: HIGH(5점), MEDIUM(3점), LOW(1점)

### 5. 참고 소스
- 학술 논문 (PubMed, Google Scholar)
- 검증된 심리 검사 도구
- 각 정보에 URL 포함

## 출력 형식
결과를 아래 형식으로 정리해주세요:

### 차원 목록
| 키 | 이름 | 이모지 | 설명 | LOW | HIGH | 근거 |
|----|------|--------|------|-----|------|------|

### 결과 유형
| # | 이름 | 이모지 | 설명 | 조건 | mood | 색상 |
|---|------|--------|------|------|------|------|

(각 결과의 상세 정보도 포함)

### 질문 예시
차원별로 2-3개씩
```

## 템플릿 B: 매칭 테스트 (Matching Test)

> 사용자에게 맞는 것(음식, 음료, 식물 등)을 추천하는 테스트용

```
[주제]: {subject_name} 매칭 테스트
(예: 커피, 와인, 위스키, 식물, 향수)

## 목적
이 주제로 **취향 매칭 테스트**를 만들려고 합니다.
사용자가 질문에 답하면, 자신에게 맞는 {subject}를 추천받습니다.

## 요청 사항

### 1. 분류 체계
- 이 주제의 공인된 분류 기준이 있나요?
  - 예: 와인의 PDO/PGI, 맥주의 BJCP, 커피의 SCA Flavor Wheel
- 전문가들이 사용하는 평가 기준
- 맛/향/특성을 구분하는 표준 프레임워크

### 2. 취향 결정 요인
- 소비자 선호도를 결정하는 주요 요인들 (심리학/마케팅 연구)
- 맛 지각 관련 과학적 요인 (쓴맛 민감도, 향 인지 등)
- 성격/라이프스타일과의 상관관계 연구

### 3. 차원(Dimension) 설계
5-6개 차원을 제안해주세요. 각 차원에 대해:
- **키(영문)**: camelCase
- **이름(한글)**: 2-4자
- **이모지**: 1개
- **설명**: 10-20자
- **양극단**: LOW ↔ HIGH
- **근거**: 왜 이 차원이 취향 구분에 중요한지

### 4. 결과 유형 (8-16개)
각 추천 결과에 대해:
- **이름**: 3-8자
- **이모지**: 1개
- **한줄 설명**: 15-30자
- **조건**: 어떤 차원 조합일 때 추천되는지 (2-3개 조건)
- **특징**: 이 유형의 맛/향/특성
- **추천 대상**: 어떤 사람에게 맞는지
- **해석**: 50-100자
- **가이드**: 즐기는 팁/활용법 50-100자
- **매치 포인트**: 추천 이유 3-4개
- **분위기(mood)**: happy, cool, excited, calm 중 하나
- **색상 제안**: 어울리는 색상

### 5. 질문 예시
각 차원당 2-3개 질문 예시:
- 질문 내용
- 3개 선택지: HIGH(5점), MEDIUM(3점), LOW(1점)

### 6. 참고 소스
- 학술 논문
- 업계 표준 (협회, 인증 기관)
- 전문 사이트
- 각 정보에 URL 포함

## 출력 형식
결과를 아래 형식으로 정리해주세요:

### 분류 체계
| 분류명 | 출처 | 설명 |
|--------|------|------|

### 차원 목록
| 키 | 이름 | 이모지 | 설명 | LOW | HIGH | 근거 |
|----|------|--------|------|-----|------|------|

### 결과 유형
| # | 이름 | 이모지 | 설명 | 조건 | mood | 색상 |
|---|------|--------|------|------|------|------|

(각 결과의 상세 정보, 매치 포인트 포함)

### 질문 예시
차원별로 2-3개씩
```

## 템플릿 C: 세부 테스트 (Detail Test)

> 메인 테스트 결과에서 세부 유형을 추천하는 테스트용

```
[주제]: {parent_subject}의 세부 테스트 - {detail_subject}
(예: 술 → 위스키, 반려동물 → 고양이 품종)

## 배경
메인 테스트({parent_subject})에서 "{result_name}" 결과가 나온 사용자를 위한
**세부 매칭 테스트**를 만들려고 합니다.

## 참고할 소스 (있다면)
- {source_url_1}
- {source_url_2}

## 요청 사항

### 1. 세부 분류 체계
- {detail_subject} 내 세부 유형/품종/종류 (8-16개)
- 각 유형의 특징과 차별점
- 공인된 분류 기준 (예: AKC 품종 기준, BJCP 맥주 스타일)

### 2. 차원(Dimension) 설계
메인 테스트와 **다른**, 세부 특화된 5-6개 축:
- **키(영문)**: camelCase
- **이름(한글)**: 2-4자
- **이모지**: 1개
- **설명**: 10-20자
- **양극단**: LOW ↔ HIGH
- **근거**: 세부 구분에 왜 중요한지

### 3. 결과 유형 (8-16개)
각 세부 유형에 대해:
- **이름**: 3-8자
- **이모지**: 1개
- **한줄 설명**: 15-30자
- **조건**: 2-3개 차원 조건
- **상세 정보 (detailInfo)**:
  - 원산지, 특징, 크기/가격대 등
  - 장점, 주의사항
  - 관리/즐기는 방법
  - 추천 대상
- **해석**: 50-100자
- **가이드**: 50-100자
- **매치 포인트**: 3-4개
- **분위기(mood)**: happy, cool, excited, calm
- **색상 제안**

### 4. 질문 예시
차원당 2-3개, 3단계 선택지 포함

### 5. 참고 소스
모든 정보에 URL 포함

## 출력 형식
[템플릿 B와 동일 + detailInfo 섹션 추가]
```

## 템플릿 D: 확장 요청 (특정 소스 포함)

> 이미 알고 있는 소스를 기반으로 리서치할 때

```
[주제]: {subject_name}

## 참고할 소스 (이것들을 포함해서 조사해주세요)
- {source_url_1}: {간단 설명}
- {source_url_2}: {간단 설명}
- {source_url_3}: {간단 설명}

## 추가 요청
위 소스들의 분류 체계를 분석하고, 다른 소스들과 비교해서
가장 적합한 프레임워크를 제안해주세요.

[템플릿 A 또는 B의 나머지 내용]
```

## 예시: 성격 테스트 - 고양이

```
[주제]: 고양이 성격 테스트

## 목적
고양이의 성격 유형을 분석하는 테스트를 만들려고 합니다.
보호자가 질문에 답하면, 고양이의 성격 유형을 알 수 있습니다.

## 참고할 소스
- Feline Five 논문: https://journals.plos.org/plosone/article?id=10.1371/journal.pone.0183455
- MCPQ-R (Monash Cat Personality Questionnaire-Revised)

## 요청 사항

### 1. 학술적 성격 모델
- Feline Five 모델의 5가지 차원 상세 분석
- 각 차원의 정의, 측정 방법
- 원본 연구의 신뢰도/타당도

### 2. 차원(Dimension) 설계
Feline Five 기반으로 5개 차원 제안:
| 키 | 이름 | 이모지 | 설명 | LOW | HIGH | 근거 |

### 3. 결과 유형 (8-12개)
고양이 성격 유형 제안:
- 각 유형별 이름, 특징, 조건
- 보호자를 위한 해석과 양육 가이드

### 4. 질문 예시
Feline Five 각 차원을 측정하는 질문 2-3개씩
3단계 선택지 포함 (항상/가끔/거의 안함)

### 5. 참고 소스
원본 논문 + 관련 후속 연구

## 출력 형식
[성격 테스트 형식으로 정리]
```

## 예시: 매칭 테스트 - 위스키

```
[주제]: 위스키 매칭 테스트

## 목적
사용자의 취향에 맞는 위스키를 추천하는 테스트를 만들려고 합니다.

## 참고할 소스
- Whisky Tasting Wheel: https://www.whiskeymasters.org/whisky-tasting-wheel
- Wishart 풍미 분류: https://whiskyanalysis.com/

## 요청 사항

### 1. 분류 체계
- 위스키 종류별 분류 (스카치, 버번, 아이리쉬, 재패니즈)
- 풍미 프로필별 분류 (피티, 스모키, 프루티 등)
- Wishart 분류 체계 상세

### 2. 취향 결정 요인
- 위스키 선호도를 결정하는 주요 요인
- 초보자 vs 마니아 취향 차이
- 음용 방식(스트레이트, 하이볼)과 취향 관계

### 3. 차원(Dimension) 설계
Wishart 분류 참고하여 5-6개 축:
| 키 | 이름 | 이모지 | 설명 | LOW | HIGH | 근거 |

### 4. 결과 유형 (10-12개)
위스키 유형별:
- 이름, 대표 브랜드
- 풍미 특징
- 어떤 사람에게 추천?
- 가격대

### 5. 질문 예시
각 차원 측정 질문 2-3개
3단계 선택지 (좋아요/보통/싫어요)

### 6. 참고 소스
학술 연구, 위스키 평가 사이트, 전문가 리뷰

## 출력 형식
[매칭 테스트 형식으로 정리]
```
