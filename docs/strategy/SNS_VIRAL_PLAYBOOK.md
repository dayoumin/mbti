# SNS 바이럴 플레이북

> **실행 시점**: 앱 완성도가 충분히 높아진 후 (Phase 2 이후)
> **선행 조건**: 테스트/투표/공유 기능 완성, 충분한 데이터 축적

---

## 1. 개요

### 1.1 목표
- SNS를 통한 신규 유입 확보
- 테스트/투표 데이터를 바이럴 콘텐츠로 재활용
- 최소 운영으로 지속적인 콘텐츠 생산

### 1.2 핵심 원칙
1. **하나의 데이터 → 여러 플랫폼**: 콘텐츠 재활용 극대화
2. **자동화 우선**: 수동 작업 최소화
3. **CTA 필수**: 모든 콘텐츠에 웹 유입 유도

## 3. 콘텐츠 유형

### 3.1 랭킹/순위 콘텐츠

**특징**: 틱톡/쇼츠에서 가장 잘 먹히는 포맷

**포맷**:
```
"이번 주 가장 인기있는 강아지 품종 TOP 5"
→ 5위부터 카운트다운 (서스펜스)
→ 1위 발표
→ "당신의 선택은? 링크에서 테스트해보세요"
```

**콘텐츠 아이디어**:
| 콘텐츠 | 업데이트 주기 | 바이럴 포인트 |
|--------|-------------|--------------|
| 주간 인기 품종 TOP 5 | 주 1회 | 순위 변동, 예상 외 결과 |
| 초보 집사 추천 TOP 5 | 월 1회 | 실용성, 저장 |
| 세대별 선택 차이 | 월 1회 | 세대 논쟁(?) 유머 |
| 지역별 인기 품종 | 월 1회 | 지역 자부심 |
| 월 유지비 순위 | 월 1회 | 현실 정보 |

**템플릿 구조**:
```
[0-3초] 훅: "1위가 이거라고?!"
[3-10초] 5위-4위 빠르게
[10-30초] 3위-2위 약간 느리게
[30-50초] 1위 발표 (서스펜스)
[50-60초] CTA: "당신의 결과는?"
```

### 3.2 VS/투표 콘텐츠

**특징**: 댓글 참여 유도, 논쟁으로 바이럴

**포맷**:
```
"고양이 vs 강아지, 첫 반려동물로 더 좋은 건?"
→ 현재 투표 결과 공개 (예: 52% vs 48%)
→ "당신의 선택은?" + 댓글 유도
→ 투표 링크
```

**콘텐츠 아이디어**:
| 콘텐츠 | 형태 | 바이럴 포인트 |
|--------|------|--------------|
| 고양이 vs 강아지 | 영상/이미지 | 클래식 논쟁 |
| 습식 vs 건식 | 영상 | 집사들 관심사 |
| 장모 vs 단모 | 이미지 | 취향 논쟁 |
| 아침산책 vs 저녁산책 | 영상 | 실생활 |
| [결과A] vs [결과B] 누가 더 | 영상 | 테스트 결과 활용 |

**템플릿 구조**:
```
[0-3초] 질문 제시
[3-15초] A 옵션 설명/이미지
[15-27초] B 옵션 설명/이미지
[27-35초] 현재 투표 결과
[35-40초] "댓글로 알려주세요!"
```

### 3.3 정보/팁 콘텐츠

**특징**: 저장/공유 유도, 실용성 강조

**포맷**:
```
"골든리트리버 키우기 전 알아야 할 10가지"
→ 빠른 리스트 or 캐러셀
→ "저장해두세요!"
→ "더 많은 정보는 프로필에서"
```

**콘텐츠 아이디어**:
| 콘텐츠 | 형태 | 바이럴 포인트 |
|--------|------|--------------|
| [품종] 키우기 전 알아야 할 N가지 | 캐러셀/영상 | 실용성 |
| 여름철 강아지 관리 체크리스트 | 카드 이미지 | 시의성 |
| 고양이 응급상황 대처법 | 캐러셀 | 저장 욕구 |
| 반려동물별 월 유지비 비교 | 인포그래픽 | 현실 정보 |
| 초보 집사 실수 TOP 5 | 영상 | 공감 |

**템플릿 구조 (캐러셀)**:
```
[1장] 훅 제목 + 대표 이미지
[2-9장] 팁 하나씩
[10장] CTA: "저장하고 프로필에서 테스트해보세요"
```

### 3.4 테스트 결과 콘텐츠

**특징**: 테스트 유입 직접 유도

**포맷**:
```
"당신이 [결과]라면..."
→ 결과 특징 재미있게 설명
→ "맞아?" + 댓글 유도
→ 테스트 링크
```

**콘텐츠 아이디어**:
| 콘텐츠 | 형태 | 바이럴 포인트 |
|--------|------|--------------|
| [결과] 유형의 특징 | 영상 | 공감, 태그 |
| [결과A] vs [결과B] 일상 비교 | 영상 | 유형 대결 |
| 가장 희귀한 결과 TOP 3 | 영상 | 희소성 |
| [결과]가 연애할 때 | 영상 | 연애 관심사 |

## 4. 콘텐츠 재활용 흐름

### 4.1 원본 → 멀티 플랫폼

```
[원본 데이터]
예: 주간 인기 강아지 품종 TOP 5
    ↓
[틱톡] 카운트다운 영상 60초
- BGM: 트렌디한 음악
- 자막: 큰 글씨, 이모지
    ↓
[유튜브 쇼츠] 동일 영상 리포맷
- 워터마크 제거/변경
- 설명란에 링크
    ↓
[인스타 릴스] 30초 압축 버전
- 1-3위만 집중
- 스토리 공유 유도
    ↓
[인스타 피드] 랭킹 카드 이미지
- 캐러셀 5장
- 저장 유도 문구
    ↓
[페이스북] 이미지 + 질문
- "1위 예상했나요?"
- 댓글 유도
```

### 4.2 주간 콘텐츠 캘린더 (예시)

| 요일 | 틱톡 | 인스타 | 유튜브 | 페이스북 |
|------|------|--------|--------|----------|
| 월 | 주간 랭킹 | - | 주간 랭킹 | - |
| 화 | VS 투표 | 릴스(VS) | - | - |
| 수 | 팁 영상 | 피드(캐러셀) | - | 투표 결과 |
| 목 | 결과 유형 | - | 팁 영상 | - |
| 금 | VS 결과 | 릴스(결과) | - | - |
| 토 | - | 피드(랭킹) | - | 주간 요약 |
| 일 | 예고 | - | - | - |

## 6. 자동화 파이프라인

### 6.1 데이터 → 콘텐츠 자동화

```
[1단계: 데이터 수집] (자동)
├── 테스트 결과 통계 (RankingService)
├── 투표 결과 (FeedbackService)
└── 메타데이터 (계절팁, 비용 등)
    ↓
[2단계: 콘텐츠 생성] (반자동)
├── 텍스트: 템플릿 기반 자동 생성
├── 이미지: Figma/Canva 템플릿
└── 영상: 템플릿 영상 + 데이터 교체
    ↓
[3단계: 검수] (수동)
├── 내용 확인
└── 플랫폼별 최적화
    ↓
[4단계: 업로드] (반자동)
├── 예약 게시 도구 활용
└── 플랫폼별 최적 시간
```

### 6.2 자동화 도구

| 단계 | 도구 | 용도 |
|------|------|------|
| 데이터 수집 | 자체 API | 통계 추출 |
| 이미지 생성 | Canva API / Figma | 템플릿 기반 |
| 영상 생성 | CapCut 템플릿 / Remotion | 자동 렌더링 |
| 예약 게시 | Later / Buffer | 멀티 플랫폼 |
| 분석 | 플랫폼 인사이트 + GA | 성과 추적 |

### 6.3 자동화 우선순위

| 단계 | 우선순위 | 설명 |
|------|---------|------|
| 1 | 템플릿 제작 | 재사용 가능한 디자인/영상 템플릿 |
| 2 | 데이터 자동 추출 | 주간 랭킹, 투표 결과 API |
| 3 | 이미지 자동 생성 | 데이터 → 이미지 파이프라인 |
| 4 | 영상 반자동화 | 템플릿 + 수동 편집 |
| 5 | 업로드 자동화 | 예약 게시 |

## 10. 공유 기능 기술 가이드

> **상세 구현 계획**: [PHASE1_SHARE_IMPLEMENTATION.md](PHASE1_SHARE_IMPLEMENTATION.md)

### 10.1 핵심 기술 스택

| 기술 | 용도 | 장점 |
|------|------|------|
| **Vercel OG** | 동적 이미지 생성 | Edge에서 800ms, Tailwind 지원 |
| **Next.js ImageResponse** | OG API 구현 | 서버 컴포넌트로 간단 구현 |
| **카카오 SDK** | 카카오톡 공유 | 한국 시장 필수 |
| **Web Share API** | 네이티브 공유 | 모바일 UX 최적 |

### 10.2 플랫폼별 이미지 비율

| 플랫폼 | 비율 | 크기 | API 파라미터 |
|--------|------|------|--------------|
| 링크 미리보기 | 1.91:1 | 1200x630 | `ratio=default` |
| 인스타/틱톡 스토리 | 9:16 | 1080x1920 | `ratio=story` |
| 인스타 피드 | 1:1 | 1080x1080 | `ratio=square` |
| 카카오톡 | 2:1 | 800x400 | `ratio=kakao` |

### 10.3 3초 훅 원칙

SNS 피드에서 스크롤을 멈추게 하려면 3초 안에 핵심 전달:

```
[상단 20%] 훅: 이모지 + 결과 이름 (눈에 띄게)
[중앙 50%] 핵심: 한 줄 설명 + 점수 시각화
[하단 30%] CTA: "나도 테스트하기 →"
```

**적용 예시:**
- 결과 카드: "🐕 골든리트리버" → "활발하고 친근한 당신!" → "테스트하기"
- VS 투표: "🐱 vs 🐕" → "현재 68% : 32%" → "투표하기"

### 10.4 저장/공유 유도 문구

| 상황 | 문구 | 효과 |
|------|------|------|
| 정보 콘텐츠 | "저장해두세요!" | 저장률 ↑ |
| 결과 공유 | "친구와 비교해보세요" | 공유율 ↑ |
| 태그 유도 | "@친구 이거 너야" | 바이럴 ↑ |
| FOMO | "이번 주만 가능!" | 즉시 행동 ↑ |

### 10.5 UTM 추적

모든 공유 링크에 UTM 자동 추가:

```
?utm_source=instagram|tiktok|kakao|facebook|copy
&utm_medium=share|story|feed
&utm_campaign=test_result|vs_poll|ranking
&utm_content={subject}_{result}
```

**추적 흐름:**
```
공유 → UTM 자동 추가 → 유입 → UTM 파싱 → 저장 → 전환 기록
```
