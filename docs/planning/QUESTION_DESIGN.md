## 2. 질문 설계 원칙

### 2.1 좋은 질문의 조건

```typescript
interface Question {
  id: string;                    // 고유 ID (예: "cat_curious_01")
  dimension: string;             // 측정 차원 (예: "호기심")
  category?: string;             // 상황 카테고리
  q: string;                     // 질문 텍스트
  options: [
    { text: string; score: number },  // 낮은 점수 (1-2)
    { text: string; score: number },  // 중간 점수 (3)
    { text: string; score: number },  // 높은 점수 (4-5)
  ];
  weight?: 1 | 2;                // 가중치 (선택)
}
```

### 2.2 설계 원칙

| 원칙 | 설명 | 예시 |
|------|------|------|
| **상황 기반** | 추상적 질문 ❌ → 구체적 상황 ✅ | "외향적이다" ❌ → "새로운 모임에서 나는..." ✅ |
| **행동 묘사** | 감정/생각 ❌ → 행동 ✅ | "사람들이 좋다" ❌ → "먼저 말을 건다" ✅ |
| **일상적** | 특수 상황 ❌ → 일상 상황 ✅ | "무인도에서..." ❌ → "주말에 나는..." ✅ |
| **양립 가능** | 한쪽만 정답 ❌ → 둘 다 합리적 ✅ | 둘 다 선택 가능해야 함 |
| **문화 적합** | 한국 문화에 맞는 상황 | 동창회, 카톡, 애견카페 등 |

### 2.3 피해야 할 패턴

```
❌ "나는 외향적인 편이다" → 자기 평가 직접 질문
❌ "창의적인 것과 현실적인 것 중 뭐가 좋나요?" → 직접적 선호 질문
❌ "무인도에 떨어지면..." → 비현실적 상황
❌ "당신은 리더입니까?" → 사회적 바람직성 편향
❌ "파티에서 많은 사람과 어울리기 vs 혼자 책 읽기" → 전형적 대조쌍 (법적 위험)
```

## 6. 실제 데이터 구조 (data.js)

### 6.1 현재 사용 중인 질문 구조

```javascript
{
  q: "질문 텍스트?",
  dimension: "차원키",   // "inssa", "adventure", "empathy", "plan", "mental" 등
  a: [
    { text: "높은 점수 선택지", score: 5 },
    { text: "낮은 점수 선택지", score: 1 }
  ]
}
```

### 6.2 실제 예시 (사람 모드)

```javascript
// 기본 질문 예시
{
  q: "시험이나 중요한 일을 앞두고 있을 때?",
  dimension: "mental",
  a: [
    { text: "할 건 했으니까! 담담하게 기다린다.", score: 5 },
    { text: "잘 준비할 게 있나? 계속 불안하다.", score: 1 }
  ]
}

// 심화 질문 예시
{
  q: "중요한 발표나 면접 전날 밤?",
  dimension: "mental",
  a: [
    { text: "긴장되긴 해도 잘 자고 일어난다. 내일은 내일의 해에서!", score: 5 },
    { text: "밤새 걱정하고 시뮬레이션을 돌린다. 잠이 잘 안 온다.", score: 1 }
  ]
}
```

### 6.3 점수 체계

| 점수 | 의미 | 사용 예시 |
|------|------|-----------|
| **5** | 해당 차원이 **높음** | 인싸력 질문에서 "먼저 말을 건다" |
| **4** | 해당 차원이 약간 높음 | 중간 수준 응답 (선택적 사용) |
| **3** | 중립 | 거의 사용하지 않음 |
| **2** | 해당 차원이 약간 낮음 | 중간 수준 응답 (선택적 사용) |
| **1** | 해당 차원이 **낮음** | 인싸력 질문에서 "조용히 있는다" |

**현재 구조**: 대부분 5점 vs 1점의 이분법 구조 사용 (2개 선택지)

### 6.4 결과 라벨 매칭 시스템

```javascript
// 결과 라벨 정의
{
  id: "passion_mansoor",
  name: "열정 만수르",
  emoji: "🔥",
  conditions: { inssa: "high", adventure: "high" },  // 조건
  desc: "일단 GO! 생각은 나중에!",
  interpretation: "...",
  guide: "...",
  mood: "excited",
  color: "bg-orange-100"
}

// 점수 레벨 판정 기준
// - high: 점수 >= 60%
// - low: 점수 <= 40%
// - medium: 그 외
```
